Варіант 5: Система управління логістичним центром (Посилки)
Створити систему управління службою доставки, яка охоплює реєстр посилок, облік кур'єрів, чергу на відправлення та історію успішно доставлених вантажів.

Що має бути в системі
Enum ParcelStatus: (Прийнято / Складується / В дорозі / Доставлено).

Інтерфейс IParcel: Визначає базові властивості посилки (ID, назва, вага, місто, статус) та метод для виведення інформації.

Клас, що представляє посилку: Реалізує IParcel (з ідентифікатором, назвою, вагою, містом призначення та статусом через enum).

Клас, що представляє кур'єра: (З ідентифікатором, ім'ям, типом транспорту та списком посилок, які він зараз доставляє).

Головний клас-контейнер служби доставки: Об'єднує всі колекції та методи для роботи з ними.

Які колекції використати (всі 4)
List — для зберігання загального реєстру всіх посилок, що коли-небудь надходили в систему.

Dictionary — для швидкого обліку кур'єрів за їхнім ідентифікатором (ключ — ID кур'єра, значення — об'єкт кур'єра).

Queue — для моделювання черги посилок на складі, що очікують навантаження (перша прийшла — перша поїхала).

Stack — для ведення історії останніх доставлених посилок (остання доставлена — зверху).

Обов'язковий функціонал (методи в класах)
Додати нову посилку: Створити об'єкт посилки, додати в загальний List та поставити в чергу Queue на складі.

Зареєструвати кур'єра: Додати кур'єра до словника Dictionary за його ID.

Відправити на доставку: Взяти наступну посилку з черги (Queue), змінити її статус на В дорозі та призначити конкретному кур'єру.

Знайти кур'єра: Знайти об'єкт кур'єра у словнику за ID та вивести список його поточних посилок.

Змінити статус посилки: Метод для оновлення enum статусу (наприклад, з В дорозі на Доставлено).

Завершити доставку: Видалити посилку у кур'єра та перемістити її в історію доставлених (Stack).

Фільтрація: Вивести всі посилки з загального списку, які прямують до певного міста.

Звітність: Методи для виведення посилок на складі (черга), активних кур'єрів та перегляду останньої доставленої посилки (через Stack.Peek()).

Вимоги до реалізації
Program.cs: Лише створення об'єктів (кур'єрів, посилок) та виклик методів для демонстрації циклу: Прийом -> Склад -> Кур'єр -> Доставка -> Історія.

Інкапсуляція: Статус посилки не повинен змінюватися просто так, лише через спеціальні методи.
